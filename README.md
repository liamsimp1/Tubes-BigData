### Berikut adalah halaman pada dashbord

Gambar ini merupakan tampilan halaman utama sistem deteksi bullying. Pada halaman ini ditampilkan identitas sistem serta gambaran umum fungsi dashboard. Halaman utama berperan sebagai pintu masuk pengguna untuk memantau dan mengakses seluruh fitur analisis data yang tersedia.
![alt text](<SS DASHBOARD BIG DATA/BIG DATA  (4).jpeg>)
Gambar ini menampilkan metrik utama sistem secara real-time, meliputi total tweet yang dianalisis, jumlah tweet dengan sentimen negatif, tingkat risiko tinggi, serta jumlah anomali CCTV. Informasi ini berfungsi sebagai ringkasan cepat untuk menggambarkan kondisi bullying terkini berdasarkan data yang masuk ke dalam sistem.
![alt text](<SS DASHBOARD BIG DATA/BIG DATA  (5).jpeg>)
Gambar ini memperlihatkan peta heatmap Indonesia yang menunjukkan persebaran risiko bullying dan anomali CCTV di berbagai wilayah. Intensitas warna pada peta merepresentasikan tingkat risiko, sehingga memudahkan pengguna dalam mengidentifikasi daerah dengan potensi bullying yang lebih tinggi dibandingkan wilayah lainnya.
![alt text](<SS DASHBOARD BIG DATA/BIG DATA  (6).jpeg>)
Gambar ini menampilkan hasil analisis sentimen tweet dalam bentuk diagram lingkaran yang terdiri dari sentimen negatif, netral, dan positif. Visualisasi ini digunakan untuk mengetahui proporsi masing-masing sentimen, serta menunjukkan dominasi sentimen negatif yang mengindikasikan adanya potensi konten bullying pada data tweet yang dianalisis.
![alt text](<SS DASHBOARD BIG DATA/BIG DATA  (8).jpeg>)
Gambar ini menunjukkan distribusi level risiko bullying dalam bentuk diagram batang. Data diklasifikasikan ke dalam beberapa kategori, yaitu risiko merah, kuning, hijau, dan aman. Visualisasi ini membantu dalam memahami tingkat keparahan risiko bullying yang terdeteksi oleh sistem.
![alt text](<SS DASHBOARD BIG DATA/BIG DATA  (7).jpeg>)
Gambar ini menampilkan grafik tren jumlah tweet per hari. Grafik ini digunakan untuk menganalisis pola temporal aktivitas bullying dari waktu ke waktu. Melalui grafik ini, dapat diketahui periode tertentu yang mengalami peningkatan atau penurunan jumlah tweet terkait bullying.
![alt text](<SS DASHBOARD BIG DATA/BIG DATA  (9).jpeg>)
Gambar ini menampilkan jumlah anomali CCTV berdasarkan lokasi tertentu, seperti toilet, kantin, lorong, kelas, dan area lainnya. Informasi ini berguna untuk mengidentifikasi lokasi yang memiliki potensi kejadian bullying lebih tinggi berdasarkan hasil deteksi anomali sistem.
![alt text](<SS DASHBOARD BIG DATA/BIG DATA  (1).jpeg>)
Gambar ini menampilkan ringkasan statistik data yang tersimpan dalam database MongoDB, meliputi distribusi sentimen, distribusi level risiko, serta kota dengan jumlah kasus terbanyak. Visualisasi ini menunjukkan bahwa data telah tersimpan dan terkelola dengan baik sebagai dasar analisis sistem.
![alt text](<SS DASHBOARD BIG DATA/BIG DATA  (2).jpeg>)
Gambar ini menampilkan contoh data mentah berupa tweet dan log CCTV yang digunakan sebagai input sistem. Sample data ini ditampilkan untuk memberikan gambaran data asli yang dianalisis serta memastikan bahwa proses analisis dilakukan berdasarkan data yang nyata.
![alt text](<SS DASHBOARD BIG DATA/BIG DATA  (10).jpeg>)
Gambar ini menampilkan keseluruhan data tweet dan CCTV log yang telah dianalisis, dilengkapi dengan fitur filter berdasarkan kota, sentimen, dan level risiko, serta pagination. Fitur ini mendukung eksplorasi data dalam jumlah besar dan memudahkan pengguna dalam melakukan pencarian data secara spesifik.
![alt text](<SS DASHBOARD BIG DATA/BIG DATA  (11).jpeg>)
Gambar ini menampilkan detail analisis setiap tweet secara individual, termasuk teks tweet, lokasi, sekolah, sentimen, level risiko, dan skor risiko. Tampilan ini menunjukkan bahwa sistem mampu melakukan analisis hingga tingkat data individu, sehingga membuktikan bahwa proses analisis berjalan secara menyeluruh dari awal hingga akhir.
